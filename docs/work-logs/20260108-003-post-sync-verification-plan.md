# 동기화 후 검증 및 정리 계획

마스터의 피드백에 따라, Git 저장소 동기화 후 발견된 문제들을 해결하고 전체적인 무결성을 검증하기 위한 계획입니다.

### 1. 루트 디렉토리 정리
- [ ] 프로젝트 루트에 생성된 불필요한 `h` 파일을 삭제합니다.

### 2. 서브모듈 전체 검증
- [ ] `git submodule status --recursive` 명령을 사용하여 모든 서브모듈(중첩 포함)의 상태를 종합적으로 확인합니다.
- [ ] `hubs-all-in-one` 서브모듈을 정밀 검사합니다.
    - [ ] 내부의 중첩된 서브모듈 구조를 파악합니다.
    - [ ] `.env.prod` 설정 파일의 위치와 구조를 확인하고 특이사항을 보고합니다.
    - [ ] 각 서브모듈이 올바른 브랜치에 있는지, 커밋되지 않은 변경사항은 없는지 확인합니다.
- [ ] 다른 최상위 서브모듈(`xrcloud-backend`, `xrcloud-frontend`, `xrcloud-nginx`)의 상태를 모두 검증합니다.

### 3. 문서 검토
- [ ] `docs/work-logs/repo_docs_audit_plan.md` 문서를 읽고 검토 범위를 명확히 파악합니다.
- [ ] 감사 계획에 따라 관련된 모든 프로젝트 문서를 현재 저장소 상태와 비교하여 정확성과 일관성을 검토합니다.
- [ ] 수정이 필요하거나 불일치하는 부분이 있으면 보고합니다.
