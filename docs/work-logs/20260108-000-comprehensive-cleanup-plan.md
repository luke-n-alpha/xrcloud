# 종합 정리 및 검증 계획

Git 동기화 이후 발견된 문제점들과 마스터의 추가 지시사항을 모두 반영한 종합 계획입니다.

### 1단계: 파일명 규칙 수립 및 적용

1.  **규칙 문서화**: `docs/work-logs/` 디렉토리에 파일명 규칙을 명시한 `README.md` 파일을 생성합니다.
    -   **규칙**: `YYYYMMDD-일련번호-주제.md` (예: `20260108-001-caret-master-plan.md`)
2.  **기존 파일명 일괄 변경**: `docs/work-logs/` 및 루트의 `work-logs/`에 있는 모든 `.md` 파일명을 새 규칙에 따라 변경합니다.

### 2단계: 저장소 정리 및 검증

1.  **불필요한 파일 삭제**: 프로젝트 루트에 생성된 `h` 파일을 삭제합니다.
2.  **서브모듈 재귀적 검증**: `git submodule status --recursive` 명령으로 모든 서브모듈의 상태를 종합적으로 확인합니다.
3.  **`hubs-all-in-one` 서브모듈 심층 검사**:
    -   내부의 중첩된 서브모듈 구조를 파악합니다.
    -   `.env.prod` 설정 파일의 위치와 구조를 확인하고 특이사항을 보고합니다.
    -   각 서브모듈의 브랜치 및 커밋되지 않은 변경사항 여부를 확인합니다.
4.  **기타 서브모듈 검증**: `xrcloud-backend`, `xrcloud-frontend`, `xrcloud-nginx`의 상태를 모두 검증합니다.

### 3단계: 문서 감사 및 기록

1.  **문서 감사 계획 실행**: `repo_docs_audit_plan.md` (파일명 변경 후) 문서를 기준으로 관련된 모든 프로젝트 문서를 현재 저장소 상태와 비교하여 검토합니다.
2.  **작업 로그 기록**: 이 모든 과정을 새 파일명 규칙에 따른 일일 로그 파일에 상세히 기록합니다.

이 계획에 대한 마스터의 승인을 받은 후, 각 단계를 순서대로 실행하겠습니다.
