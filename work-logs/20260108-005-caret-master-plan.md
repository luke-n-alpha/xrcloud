본 문서는 XRCloud 프로젝트의 Git 저장소 구조를 재정비하고, 오픈소스 공개를 준비하며, AI 에이전트와의 협업 환경을 구축하기 위한 종합 계획을 정의합니다. 모든 작업은 단계별(Phase)로 나누어 체계적으로 진행합니다.

- [x] **1-2. 서브모듈 역할 파악:** 각 서브모듈(`frontend`, `backend`, `nginx`, `hubs-all-in-one`)의 `README.md`를 읽고 개별 역할 및 사용법을 분석합니다. (완료)
- [x] **1-3. 신규 원격 저장소 URL 확인:** 변경된 원격 저장소 URL 목록을 마스터께 확인받습니다. (완료)
- [x] **1-4. URL 변경 계획 수립:** 마스터의 지시에 따라 현행 `luke-n-alpha` 저장소 URL을 그대로 사용하기로 결정했습니다. URL 변경 작업은 불필요합니다. (완료)

**목표:** 오픈소스 공개 시 발생할 수 있는 보안 위협을 사전에 제거합니다.

- [x] **2-1. 민감 정보 스캔:** 모든 저장소의 코드에서 하드코딩된 API 키, 비밀번호, 개인 서버 주소 등 민감 정보를 검색합니다. (완료)
- [x] **2-2. 환경 변수 설정 강화:** `.env.sample` 파일을 표준화하고, 실제 `.env` 파일이 커밋되지 않도록 `.gitignore` 설정을 강화합니다. (진행 중)
- [ ] **2-3. 보안 점검 결과 보고:** 발견된 보안 취약점과 수정 계획을 마스터께 보고하고 승인을 받습니다.

## Phase 4: 루트 저장소 정리 및 통합 (Integration)
**목표:** 모든 서브모듈의 변경사항을 취합하여 최상위 프로젝트를 안정적인 상태로 만듭니다.

- [ ] **4-1. 루트 저장소 파일 추가:** `xrcloud` 루트 저장소의 추적되지 않던 파일들(`.gitignore`, `README.md` 등)을 `git add`로 추가합니다.
- [ ] **4-2. 서브모듈 업데이트 반영:** 업데이트된 모든 서브모듈의 커밋 정보를 루트 저장소에 반영합니다.
- [ ] **4-3. 최종 통합 커밋:** 모든 변경사항을 담아 "chore: Refactor project structure for open source"와 같은 명확한 메시지로 첫 커밋을 수행합니다.

## Phase 5: AI 협업 환경 구성 및 문서화 (AI Collaboration)
**목표:** AI 에이전트가 프로젝트를 쉽게 이해하고 기여할 수 있는 환경을 구축합니다.

- [ ] **5-1. 표준 실행 스크립트 개선:** 프로젝트 빌드 및 실행을 위한 `run.sh`, `build.sh` 등의 스크립트를 검토하고, 누구나 쉽게 사용할 수 있도록 개선합니다.
- [ ] **5-2. AI 컨텍스트 문서 작성:** `.agents/context` 디렉터리에 프로젝트의 아키텍처, 주요 기술 스택, 개발 환경 설정 방법을 요약한 문서를 추가합니다.
- [ ] **5-3. 기여 가이드라인 작성:** `CONTRIBUTING.md` 문서를 생성하여 프로젝트에 기여하는 방법을 안내합니다.
